# Deploy Website Changes

Commit current changes, merge to main, and push to deploy via GitHub Pages.

## Instructions

1. Run `git status` and `git diff --stat` to review what will be committed.
2. Stage only relevant files — never stage `.DS_Store` files or sensitive files.
3. Write a concise commit message summarizing the changes.
4. If on a feature branch, merge to `main` first.
5. Push `main` to origin.
6. Inform the user that GitHub Pages will auto-deploy and the site will be live at www.joselazoflores.com shortly.
